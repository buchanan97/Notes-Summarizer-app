The blob is there again and it continues to move from where it was when you left it. When you start Squeak for the rst time, the Squeak virtual machine loads the image le that you provide. This le contains a snapshot of a large number of objects, including a vast amount of pre-existing code and a large number of programming tools (all of which are objects). As you work with Squeak, you will send messages to these objects, you will create new objects, and some of these objects will die and their memory will be reclaimed ( i.e., garbage-collected).
