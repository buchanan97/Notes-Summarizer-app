The main entry point: File Reference While File System is based on multiple concepts and classes such as File System ,Path and File Reference .File Reference is the most important for an end-user. File Reference offers a set of operations to manipulate les. So far, we have seen some basic operations. This section covers the more elaborated operations. At the design level, a le reference ( File Reference ) combines two low-level entities: a path ( Path) and a lesystem ( File System ) into a single object which provides a simple protocol to manipulate and handle les. A File Reference implements many operations of File System (both are largely polymorphic), but without the need to track paths and lesystem separately.
